# Release Checklist

- [ ] `RL-01` Version, candidate revision, artifact digests, previous release, and target environment are immutable/known.
- [ ] `RL-02` Release scope and changes since prior release are complete and traceable.
- [ ] `RL-03` All applicable G0–G7 evidence is current for the release candidate.
- [ ] `RL-04` Dependencies, configuration, secrets interfaces, flags, and environment prerequisites are verified.
- [ ] `RL-05` Data/contract migrations have order, compatibility window, backup, validation, and rollback.
- [ ] `RL-06` Deployment stages have owners, checkpoints, health signals, stop conditions, and communication.
- [ ] `RL-07` Rollback triggers, authority, commands/steps, data recovery, and verification are rehearsed where required.
- [ ] `RL-08` Security, privacy, performance, capacity, observability, support, and accepted risks are reviewed.
- [ ] `RL-09` Release notes, runbooks, user/operations docs, and post-release checks are complete.
- [ ] `RL-10` Human authorization exists for the exact candidate; outcome and post-flight will be recorded.
