# Pre-flight Checklist

- [ ] `PF-01` Repository root, branch, `HEAD`, default branch, remote status, and worktree status are recorded.
- [ ] `PF-02` Uncommitted changes are classified by owner; unrelated or unknown changes are protected.
- [ ] `PF-03` Current-state and target-state sources are classified under the source-of-truth hierarchy.
- [ ] `PF-04` Foundation drift was checked for architecture, terminology, rules, templates, constitution, and ownership.
- [ ] `PF-05` Dependency drift was checked for interfaces, events, contracts, packages, ownership edges, and consumers.
- [ ] `PF-06` Any detected drift was synchronized and validated before work continued.
- [ ] `PF-07` Objective, in-scope/out-of-scope work, affected modules, and success criteria are explicit.
- [ ] `PF-08` Required workflows, agents, reviews, risk level, and gates are identified.
- [ ] `PF-09` Each agent context package is minimal, sufficient, revision-bound, and contains an output contract.
- [ ] `PF-10` Blocking unknowns/conflicts are resolved or escalated; G0 status is recorded.
