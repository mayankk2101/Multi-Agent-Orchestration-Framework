# Business Rules Checklist

- [ ] `BR-01` Each rule traces to an authorized source and an accountable owner.
- [ ] `BR-02` Preconditions, outcomes, exceptions, and invariants are stated for each rule.
- [ ] `BR-03` Precedence and boundary cases are defined where rules interact or overlap.
- [ ] `BR-04` Duplicates and contradictions are detected across specs, code, tests, and terminology.
- [ ] `BR-05` Each rule has exactly one canonical definition; no competing authority remains.
- [ ] `BR-06` Interpretation is deterministic; no rule depends on implicit or conversational knowledge.
- [ ] `BR-07` Each rule outcome is testable through observable behavior.
- [ ] `BR-08` A stable rule ID is assigned to every normative rule.
- [ ] `BR-09` Missing-case and exception questions are recorded rather than assumed resolved.
- [ ] `BR-10` No rule is marked valid from conversation alone without evidence.
