# Platform Templates

These are the fixed output shapes for framework artifacts. Authors and reviewers fill a template without adding, removing, or reordering its normative sections.

## Index

- [ARCHITECTURE_REVIEW_TEMPLATE.md](ARCHITECTURE_REVIEW_TEMPLATE.md) — independent architecture review of a frozen candidate.
- [BUSINESS_RULE_VALIDATION_TEMPLATE.md](BUSINESS_RULE_VALIDATION_TEMPLATE.md) — business-rule authority, consistency, and testability validation.
- [CONSISTENCY_REVIEW_TEMPLATE.md](CONSISTENCY_REVIEW_TEMPLATE.md) — contradiction, terminology, and cross-reference review.
- [DECISION_RECORD_TEMPLATE.md](DECISION_RECORD_TEMPLATE.md) — approved architecture or policy decision.
- [DEPENDENCY_REVIEW_TEMPLATE.md](DEPENDENCY_REVIEW_TEMPLATE.md) — module, contract, event, and package dependency review.
- [IMPLEMENTATION_PLAN_TEMPLATE.md](IMPLEMENTATION_PLAN_TEMPLATE.md) — ordered, reversible, testable implementation plan.
- [IMPLEMENTATION_REPORT_TEMPLATE.md](IMPLEMENTATION_REPORT_TEMPLATE.md) — engineer report of scoped changes, checks, and evidence.
- [MODULE_SPEC_TEMPLATE.md](MODULE_SPEC_TEMPLATE.md) — implementation-independent module specification.
- [PERFORMANCE_REVIEW_TEMPLATE.md](PERFORMANCE_REVIEW_TEMPLATE.md) — independent performance and scalability review.
- [PR_REVIEW_TEMPLATE.md](PR_REVIEW_TEMPLATE.md) — pull-request review record.
- [RELEASE_REPORT_TEMPLATE.md](RELEASE_REPORT_TEMPLATE.md) — release candidate, gate evidence, and rollback package.
- [REQUIREMENTS_REGISTER_TEMPLATE.md](REQUIREMENTS_REGISTER_TEMPLATE.md) — scoped, traceable requirements and acceptance criteria.
- [RISK_ASSESSMENT_TEMPLATE.md](RISK_ASSESSMENT_TEMPLATE.md) — authorized residual-risk acceptance.
- [SECURITY_REVIEW_TEMPLATE.md](SECURITY_REVIEW_TEMPLATE.md) — independent security and privacy review.
- [TEST_PLAN_TEMPLATE.md](TEST_PLAN_TEMPLATE.md) — QA test plan, coverage matrix, and execution report.
- [VALIDATION_REPORT_TEMPLATE.md](VALIDATION_REPORT_TEMPLATE.md) — independent quality-validation report.

## File Conventions

Framework template files use `UPPER_SNAKE_CASE.md`. Checklists, workflows, and agent contracts use `lowercase-kebab-case.md`. Knowledge indexes keep their existing names. These conventions document current filenames and do not rename any file.
